# NeroTech privacy & telemetry disclosure

NeroTech includes optional, anonymous error reporting so that crashes and bugs caused by the
mod can be found and fixed. This page is the full disclosure required by CurseForge's
moderation rules for mods that use an external analytics/error service, and it documents how
the system is designed to comply with the GDPR (EU) and POPIA (South Africa).

> **Note:** error reporting only becomes active once NeroTech's own Sentry DSN is configured
> in the build (`NeroTechTelemetry.DSN`). While that field is blank, NeroTech sends **nothing**,
> regardless of the settings below.

## What is collected (error reporting)

When an error **caused by NeroTech code** occurs, a report is sent to
[Sentry](https://sentry.io) (hosted in the **EU** — `ingest.de.sentry.io`) containing:

- the stack trace of the error and the error message (file paths containing your
  operating-system account name are masked before sending),
- the NeroTech version, the loader (NeoForge / Forge / Fabric) and its version, the Minecraft
  version, and whether the game is a client or dedicated server,
- the operating system name/version and Java version,
- a couple of the mod's own configuration flags (e.g. whether the Fusion meltdown and
  per-player pollution attribution options are enabled), so a crash that only happens at
  non-default settings can be reproduced,
- the list of **other loaded mods** (their public ids and versions only), so a crash caused by
  a mod conflict can be identified,
- a random event ID generated by Sentry for the individual report.

Reports are de-duplicated and hard-capped at 10 events per game session.

## What is NOT collected

- **No IP address** is stored (`sendDefaultPii` is disabled; Sentry discards the sender IP).
- **No username, player UUID, or machine/host name.**
- **No world data, chat, coordinates, inventories, or gameplay analytics** of any kind.
- **No advertising or tracking identifiers** — there is no persistent ID linking reports from
  the same player or machine.
- Errors caused by other mods or by Minecraft itself are filtered out and never sent.

## How to opt out

Set the following in `config/nerotech.properties` (created on first launch):

```properties
telemetryEnabled=false
```

This is a client-local switch (not server-synced) and covers everything error reporting sends.
The change takes effect on config reload or next launch; nothing is sent while disabled, and
nothing is ever sent before the config — and therefore your choice — has been loaded.

## Per-player pollution attribution (opt-in, off by default)

Separately from telemetry, NeroTech's **pollution** system is **regional and aggregate by
default and stores no player data**. A server may opt in to per-player attribution by setting
`pollutionPerPlayerAttribution=true`; only then does NeroTech store, in the world save:

- the **UUID** of the player who placed a machine (never their name), and a per-UUID pollution
  total with a last-updated day stamp.

This data is **minimised** (UUIDs only), **retention-pruned** after
`pollutionAttributionRetentionDays`, and registered with Neroland Core's shared data-erasure
hook — so a single erase request (or Core's inactivity sweep) purges a player's NeroTech data
alongside every other Neroland mod. With the option off (the default), none of this is stored.

## Legal basis and your rights (GDPR / POPIA)

Error reports are processed on the basis of **legitimate interest** (GDPR Art. 6(1)(f); POPIA
s11(1)(f)): keeping the mod functional and crash-free, using the minimum data necessary. They
are engineered to contain no directly identifying personal information, are stored in the EU,
and are retained only for Sentry's standard retention period before automatic deletion.

To exercise any data-subject right (access, deletion, objection), contact
**[info@neroland.co.za](mailto:info@neroland.co.za)** with the approximate date/time of the
crash so the matching event can be located and removed. Sentry acts as a data processor; see
the [Sentry privacy policy](https://sentry.io/privacy/) and
[data processing addendum](https://sentry.io/legal/dpa/).

## Changes to this policy

If a future version of NeroTech ever collects additional data, this page will be updated
**before** that version is released, the change will be called out in the changelog, and the
same opt-out (`telemetryEnabled = false`) will continue to cover everything error reporting
sends.
